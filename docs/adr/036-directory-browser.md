# ADR 036: Directory Browser for Session Creation

## Status

Accepted

## Context

ポータルページからセッションを作成する際、ディレクトリパスを手入力する必要があった。これには以下の問題があった:

1. **入力ミス**: パスのタイプミスが発生しやすい
2. **ディレクトリ構造の把握**: 利用可能なディレクトリを事前に知っている必要がある
3. **モバイルでの操作性**: キーボードでのパス入力が煩雑

また、任意のディレクトリを指定できることはセキュリティ上のリスクとなるため、許可されたディレクトリのみからセッションを作成できる仕組みが必要だった。

## Decision

**ディレクトリブラウザ UI + 許可リスト制限**のアーキテクチャを採用する。

### アーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│  Portal Page                                                │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  Directory Browser                                     │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │  📁 ~/projects (allowed_directories[0])         │  │  │
│  │  │  📁 ~/work     (allowed_directories[1])         │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  │           ↓ click                                      │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │  📁 project-a                                   │  │  │
│  │  │  📁 project-b                                   │  │  │
│  │  │  📁 project-c                                   │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  │           ↓ select                                     │  │
│  │  [Create Session: ~/projects/project-a]               │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
         │                              │
         ▼                              ▼
┌──────────────────┐         ┌──────────────────────┐
│  GET /api/       │         │  POST /api/sessions  │
│  directories/    │         │  {dir: "..."}        │
│  list            │         └──────────────────────┘
└──────────────────┘
```

### 設定オプション

```yaml
directory_browser:
  enabled: true
  allowed_directories:
    - ~/projects
    - ~/work
    - /var/www
```

### API エンドポイント

| エンドポイント | メソッド | 説明 |
|---------------|---------|------|
| `/api/directories` | GET | 許可されたベースディレクトリ一覧 |
| `/api/directories/list` | GET | サブディレクトリ参照 (`baseIndex`, `path` パラメータ) |
| `/api/directories/validate` | POST | パス検証 |

### セキュリティ対策

1. **許可リスト制限**: `allowed_directories` に指定されたディレクトリ配下のみ参照可能
2. **パストラバーサル防止**: `isRelativePathSafe()` で `..` などの危険なパスを拒否
3. **シンボリックリンク検証**: リンク先が許可ディレクトリ内かを確認
4. **隠しファイル除外**: `.` で始まるファイル/ディレクトリを非表示

## Consequences

### Positive

- **操作性向上**: クリックベースでディレクトリを選択可能
- **セキュリティ強化**: 許可リストにより意図しないディレクトリへのアクセスを防止
- **モバイル対応**: タップ操作でディレクトリをナビゲート可能
- **ユーザビリティ**: チルダ展開 (`~`) 対応で直感的なパス表示

### Negative

- **設定必須**: `allowed_directories` を設定しないと機能しない
- **UI 複雑化**: ポータルページの複雑さが増加
- **追加 API**: 3つの新規エンドポイントが必要

## Implementation Details

### ファイル構成

```
src/daemon/
├── directory-browser.ts    # ディレクトリ操作ロジック
├── api-handler.ts          # API エンドポイント追加
└── portal.ts               # UI 追加

src/config/
└── types.ts                # DirectoryBrowserConfig 型

src/utils/
└── path-security.ts        # パス検証ユーティリティ
```

### 主要関数

| 関数 | 説明 |
|------|------|
| `getAllowedDirectories()` | 許可ディレクトリ一覧を取得 |
| `listSubdirectories()` | サブディレクトリを列挙 |
| `validateDirectoryPath()` | パスが許可範囲内か検証 |
| `expandTilde()` / `contractTilde()` | チルダ展開/収縮 |
| `isRelativePathSafe()` | 相対パスの安全性検証 |

### UI フロー

1. ポータルページ読み込み時にディレクトリブラウザを表示
2. 許可されたベースディレクトリを一覧表示
3. ディレクトリをクリックでサブディレクトリを展開
4. 「ここでセッション作成」ボタンでセッション作成
5. 作成成功で自動的にセッションページへ遷移

## Notes

- `allowed_directories` が空または未設定の場合、ディレクトリブラウザは非表示
- シンボリックリンクは許可ディレクトリ内を指す場合のみ表示
- ディレクトリのソートはアルファベット順（大文字小文字区別なし）
